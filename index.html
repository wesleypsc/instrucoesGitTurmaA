TRABALHANDO COM O GITHUB ATRAVÉS DO PROMPT DE COMANDO

Comandos e instruções passo-a-passo para utilização do Git e GitHub

INSTALAÇÃO DO GIT

Para instalação do GIT, é necessário acessar o endereço http://git-scm.com, efetuar o download e executar o instalador.

A instalação em si do git, é muito simples. Avançar, avançar e avançar.

CRIAR REPOSITÓRIO NO GITHUB

É necessário acessar o GitHub e criar um repositório onde os arquivos ficarão salvos.

Este passo pode ser feito antes mesmo de instalar o GIT.

CONFIGURAR NOME E E-MAIL NO GIT

Nesta etapa é necessário configurar um nome de exibição e um e-mail para o GIT.

Com o Git Instalado, você vai abrir o prompt de comando, navegar até a pasta do projeto e executar os seguintes comandos:

git config --global user.name "SEU NOME"

git config --global user.email "E-MAIL CONFIGURADO NO GITHUB"

INICIAR E VERIFICAR O STATUS DO REPOSITÓRIO

Executar os comandos a seguir para iniciar o repositório na pasta do projeto e em seguida, verificar o status dos arquivos.

git init

git status

ADICIONAR OS ARQUIVOS NO REPOSITÓRIO "TEMPORÁRIO"

Executar o comando a seguir para adicionar os arquivos que deseja enviar ao repositório na nuvem.

git add .

Pode ser utilizado o comando "git add nome_do_arquivo.ext" para adicionar arquivos individualmente.

EFETUAR O COMMIT DOS ARQUIVOS

Executar o comando a seguir para efetuar o commit no git.

git commit -m "COMENTÁRIO DO COMMIT"

VINCULAR O PROJETO COM O GITHUB E ENVIAR OS ARQUIVOS PARA O REPOSITÓRIO NA NUVEM

Primeiro, executar o comando abaixo para vincular o diretório ao repositório no GitHub.

git remote add origin LINK DO REPOSITÓTIO

Exemplo: git remote add origin https://github.com/wesleypsc/ExemploHTML

Para isso é necessário acessar o seu usuário do GitHub e verificar o link do repositório, conforme imagem abaixo.

Em seguida, executar o comando de push para enviar os arquivos ao repositório vinculado anteriormente.

git push -u origin master

Nestes passos pode ser solicitado o usuário e senha do GITHUB!

Caso necessite substituir os arquivos já existentes do repositório, utilize o parâmetro --force

Exemplo: git push --force -u origin master
